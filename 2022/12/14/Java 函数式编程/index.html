函数式接口

一个接口中有且仅有一个方法定义，这样的接口可以称为 函数式接口，但是它可以有多个非抽象方法的接口。

可以通过 @FunctionalInterface 注解来标志一个函数式接口，不过不是必须的，在注解源码的注释中提到了

“However, the compiler will treat any interface meeting the definition of a functional interface as a functional interface regardless of whether or not a FunctionalInterface annotation is present on the interface declaration.”

“但是，无论接口声明中是否存在FunctionalInterface注释，编译器都会将满足功能接口定义的任何接口视为功能接口。”

函数式接口的实例可以使用 lambda 表达式、方法引用或构造函数引用来创建

Stream常用操作以及原理探索

Stream 是 jdk1.8 中加入的，可以很方便且高效的对集合进行一些操作，比如求和，排序，筛选等等

Stream 只操作数据，不会改变数据源，且操作具有延时性，如果一个流操作中没有终端操作，那么其中间操作不会执行，中间操作可以有任意个，但终端操作有且只能有一个

简单理解终端操作和中间操作

终端操作：没有返回值或者返回值不是一个流，那么不可以在调用流的相关方法

  • 如 foreach() ，没有返回值，所有不可以在调用其他方法，即为终端操作

中间操作：有返回值且是一个流，可以继续调用其他方法

  • 如 map()，返回了一个新的 Stream ，那么可以继续调用其他方法，即为中间操作

流的创建方式

Stream 类本身提供了几种创建流的静态方法 of(),generate() 等等

不过一般都是通过某个集合来生成流，毕竟这玩意就是用来操作集合的。也可以通过一个数组生成

Arrays.asList() 可以生成一个 ArrayList ，不过注意，这个 ArrayList 不是我们常用的 java.util 包里的，而是 Arrays 的一个私有静态内部类

还可以通过一个文件生成流等，生成流的方式很多，这里不过多赘述

顺序流，并行流

顺序流，由一主线程按照元素定义的顺序或者链式调用的各种操作的顺序依次执行

并行流，很清楚了和顺序流相反，多个线程并行执行也就是说执行的顺序是乱序的

分类

按类型，流又可按照有状态/无状态的中间操作和/短路/非短路的终端操作分为四类

  • 有状态，必须等到前面的中间操作都完成之后，才能进行的操作叫有状态的中间操作

    • 比如，sorted()，只能等到前面的操作都执行完了之后，才能对元素进行排序

  • 无状态，无效等到前面的数据完成，来一个我处理一个

    • 比如，map()，执行时，无需等待前面的操作，元素流到这一步直接处理

  • 短路，不用将流中的元素全部处理完，只要有满足条件的那么直接返回，剩下的元素不会在判断，直接返回判断结果

    • 比如，allMatch()，全部满足条件才返回 true，只要有一个不满足直接 false，后面不管还有没有元素都不在进行判断，有点类型逻辑与 &&

  • 非短路，与短路相反，需要将元素全部处理完之后，才会返回结果

    • 比如，sum()，和 sorted() 一样，只有元素都处理完了之后，才能进行求和

combiner 在顺序流中不执行是因为只有在并行流中才有组合的概念

在并行流中会将元素拆开使用多个线程并行执行 accumulator ，如果最终的结果类型和实现的类型不一致，则需要通过一个中间函数(combiner)对结果进行组合并转换成对应的预期类型(这一点在方法参数上有所体现)，但是在顺序流中并不会对流中的元素进行拆分执行，所以自然也就不需要进行组合。再者如果累加的结果的类型和实现的类型一致，则并不需要这个组合器，因为这只是从一个类型的值，变成另一个相同类型的值。

Optional 好比一个容器用来保存其实现类型的元素，又或者只保存了一个 null

方法介绍

isPresent,ifPresent

isPresent() 判断当前 Optional 包装的元素是不是 null

ifPresent() 如果元素不存在，不作任何操作，否则直接使用提供的 Consumer 函数进行消费

get,orElse,orElseGet,orElseThrow

get() 获取元素，为空直接抛出 NoSuchElementException 异常，所有使用 get() 时，要确保元素存在

orElse() 也是获取元素，但可以提供一个默认值，用于在元素为空时返回

orElseGet() 同样是获取元素，可以提供一个 Supplier 函数，在元素为空时返回其结果

orElseThrow() 也是获取元素且可以提供一个 Supplier 函数，但是 orElseThrow() 提供的供给型函数是用于产生异常而不是提供元素

map,flatMap,filter

map()，如果元素为空，会创建一个空的 Optional，否则使用提供的 Function 函数处理元素

flatMap()，在元素为空时也会创建一个空的 Optional，但是和 map() 的区别在于 Function 的返回值，flatMap() 要求提供的函数的返回值是一个 Optional，而 map() 中则是对结果进行了再次包装

filter()，使用时，如果元素为空，返回当前 Optional 实例，否则使用传入的 Predicate 函数进行判断，满足条件直接 return this，否则返回一个空实例
