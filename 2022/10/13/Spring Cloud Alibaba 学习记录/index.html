什么是Spring Cloud和 Spring Cloud Alibaba

Spring Cloud是一系列分布式框架的集合，是一种规范！

将不同公司生产的不同组件，以Spring Boot的风格进行集成，开箱即用！

需要什么就通过Spring Boot整合

Spring Cloud Alibaba 致力于提供微服务开发的一站式解决方案。此项目包含开发分布式应用服务的必需组件，方便开发者通过 Spring Cloud 编程模型轻松使用这些组件来开发分布式应用服务。

依托 Spring Cloud Alibaba，您只需要添加一些注解和少量配置，就可以将 Spring Cloud 应用接入阿里分布式应用解决方案，通过阿里中间件来迅速搭建分布式应用系统。

Sentinel 熔断限流

雪崩效应

多个存在调用关系的服务中有一个服务出现问题，而导致其他的无法调通，造成阻塞，并且会不断扩大，导致整个服务链崩溃。

解决方案

  • 设置线程超时

  • 设置限流

  • 设置服务熔断器 Sentinel、Hystrix

    • 调用服务时，如果成功，熔断器会一直处于关闭状态。一旦出现调用失败的情况，熔断器会打开限制访问，然后会在一定时间内再次尝试调用服务(这时熔断器处于半开状态，允许一次调用)，直到服务调用成功。就好比，游戏断线重连/保险丝烧断保护电路。

    • 降级 -> 功能降级，应对自身
    • 限流 -> 限制访问
    • 熔断 -> 断开链接，抛出异常，应对外部系统

降级

简书 Sentinel降级

Sentinel熔断降级会在调用链路中某个资源出现不稳定状态时（例如调用超时过异常比例升高），对这个资源的调用进行限制，让请求快速失败，避免影响到其他的资源而导致级联错误。

当资源被降级后，在接下来的降级时间窗口之内，对该资源的调用都会自动熔断（默认行为是抛出DegradeException）

  • 慢调用比例

    • 下图规则的意思为：在1000ms的时间段中，至少有五个请求，并且请求响应时间超过200ms的比例达到了1：1，那么就会触发熔断，熔断时间为5秒钟。

  • 异常比例

    • 下图规则的意思为：在1000ms的时间段中，至少有五个请求，当异常比例达到20%的时候产生触发熔断，熔断时间为5s

  • 异常数

    • 下图规则的意思为：在1000ms的时间段中，至少有五个请求，当达到指定的异常数阈值时，服务就会发生熔断，熔断时间为5s

RocketMQ 消息中间件

下载

注意：下载 Binary 这栏的

应用场景

  • 应用解耦

    当要调用远程系统时候，当存在订单系统和库存系统时，订单系统下单，库存系统需要收到订单后库存减一，这时候如果系统宕机，会造成订单丢失，吧订单消息发入mq，库存系统再去mq消费，就能解决这一问题。

  • 异步消费

    传统的模式：用户下单—>邮件发送—>短信提醒，三个步骤全部完成，才能返回用户消费成功，因为后面两个步骤完全没有必须是当前时间完成，可以用户下单成功后，直接发送给mq，返回给用户消费成功，之后邮件发送和短信提醒，可以其他时间段来消费发送给用户。

  • 流量削峰

    大型双11活动时候，0点有上亿并发，这时候数据库并不能承载那么大的数据冲击，而专门为高并发设计的mq可以承受住海量的请求，发送给mq，存储成功后，再消费。
