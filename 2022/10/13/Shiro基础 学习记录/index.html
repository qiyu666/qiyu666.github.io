Shiro基础 学习记录

本文最后更新于：2022年10月15日 晚上

官网

主流 Java 安全框架，不依赖任何容器，可以运行在 Java SE 、Java EE 项目中。

主要用于是对用户访问进行身份认证、授权、会话管理以及加密等。

核心组件

用户、角色、权限关系：

即，给用户指定角色，给角色指定权限

  • UsernamePasswordToken

    • 用于封装用户登录信息，使用其登录信息来创建令牌Token

  • SeurityManager

    • 核心部分，负责安全认证和授权 🔴

  • Suject

    • Shiro 的一个抽象概念，即包含了用户信息

  • Realm

    • 开发者的自定义模块，根据需求，将验证和授权的逻辑写在这其中

  • AuthenticationInfo

    • 用户信息集，用于认证

  • AuthorizationInfo

    • 角色权限信息集，用于授权

  • DefaultWebSecurityManager

    • 安全管理器，上文中的 Realm 必须注入其中，才可生效使用

  • ShiroFilterFactoryBean

    • 过滤器工厂，Shiro的基本运行机制是开发者定制规则，然后交给框架执行，具体执行操作就由这个 Api 创建的一个个 Filter 对象来完成

运行机制

整合 Spring Boot

  • 导入依赖包

  • 自定义 Shiro 过滤器

    • AuthorizationInfo --> 用于授权
    • AuthenticationInfo --> 用于认证

  • 配置

    • 先将自定义 Realm （AccountRealm）注入到 IoC 容器中，再使用 @Qualifier 从容器中取出来，注入到 DefaultWebSecurityManager 中，再同理将其注入到 ShiroFilterFactoryBean 中

  • 编写认证和授权规则

    认证过滤器

    • anon：无需认证
    • authc：必须认证
    • authcBasic：需要通过 HTTPBasic 认证
    • user：不一定需要认证，被 Shiro 记录过的，比如记住我功能

    授权过滤器

    • perms：必须拥有某个权限才能访问
    • role：必须拥有某个角色才能访问
    • port：请求的端口必须是指定的才能访问
    • rest：请求必须基于 Restful，POST、PUT、GET、DELETE
    • ssl：必须是安全的 URL 请求，协议必须是 HTTPS

认证

  • 在 ShiroConfig 中设置访问权限

  • 设置 login 页面跳转

  • 处理登录逻辑

    • 先通过 SecurityUtils 的 getSubject() 方法来获得 Suject ，因为这里面包含了用户信息
    • 通过用户输入的账号密码生成 token
    • 使用 subject 的 login() 方法来进行身份认证（可以设置断点进行查看，这里会进入自定义的 Realm 中进行校验）
      • 如果账号密码都正确，将进入首页 index
      • 如果账号不正确，会抛出 UnknownAccountException 异常
      • 如果账号正确，密码有误，会抛出 IncorrectCredentialsException 异常
    • 通过 try...catch 来捕获异常

授权

  • 在 Realm 中编写授权逻辑

    • 先通过 SecurityUtils 的静态方法 getSubject() 来获得 Suject ，因为这里面包含了用户信息

    • 通过 Suject 的 getPrincipal() 方法来获取用户信息

      • 这里因为是在进行用户认证之后，用户信息已经通过 SimpleAuthenticationInfo 存入了 shiro 中

    • 将用户的角色信息和权限信息放入 SimpleAuthorizationInfo 中
