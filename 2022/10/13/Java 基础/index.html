方法

什么是方法

语句的集合，他们在一起执行一个功能

  • 方法是解决一类问题的步骤的有序组合
  • 方法包含于类或对象中
  • 方法在程序中被创建，在其他地方被引用

设计原则：其本意为功能块，用于实现某个功能的代码块的集合，设计时最好保持其原子性，即 一个方法只完成一个功能，利于扩展

方法定义

方法的语法结构

  • 修饰符，可选的，告诉编译器如何调用该方法，定义方法的访问类型
    • static：定义为静态方法，可直接通过类名进行调用
    • final：其修饰的方法，不可被重写
  • 返回值类型：方法可能会有返回值，void 表示没有返回值，无须使用 return 返回任意东西，此外，如 int String double... 都 必须 使用 return 返回对应的返回值
  • 方法名：方法实际名称，和参数列表共同构成方法签名
    • 参数列表：方法参数类型、顺序和参数个数
  • 参数类型：参数就好比一个占位符，当方法被调用时，必须传入对应的参数值类型的数据，当然这个参数是可选的，方法也可以没有参数
    • 形式参数：简称 形参 ，用于接收外界调用方法是传入的数据
    • 实际参数：简称 实参 ，调用方法时实际传入的参数
  • 方法体：定义方法具体功能

静态方法和非静态方法有什么区别

  • 静态方法不能重写，如果子类存在和父类相同的静态方法，那么父类这个方法是相对于子类所隐藏，非静态方法可以被重写

  • 静态方法调用只和左边定义的数据类型有关，而非静态方法不受影响

  • 静态方法属于类所有，类实例化前即可使用。非静态方法必须通过new出来的对象调用 🔴

  • 非静态方法可以访问类中的任何成员，静态方法只能访问类中的静态成员。且静态方块中不能调用非静态方法。

  • static内部只能出现static变量和其他static方法。而且static方法中还不能使用this、super关键字，因为它是属于整个类；

  • 静态方法效率上要比实例化高，但不能自动进行销毁，而实例化的则可以做销毁；

  • 静态方法和静态变量创建后始终使用同一块内存，而使用实例的方式会创建多个内存。

方法重载 && 方法重写

菜鸟教程 Java 重写(Override)与重载(Overload)

重载，即在一个类中，有相同名称的方法，但形参不同

重载规则

  • 方法名 必须相同
  • 参数列表 必须不同
  • 方法返回值类型可相同也可不同
  • 仅仅是返回值类型不同不足以成为方法的重载

重写，即子类对父类的允许访问的方法的实现过程进行重新编写

重写规则

外壳不变，核心重写！

  • 需要有继承关系

  • 只能重写 public/default 修饰的方法

  • 返回值、形参和方法名都 一定不能改变

  • 方法体重写

  • 修饰符可以扩大，但不能缩小

    • public > protected > default > private
    • 比如父类方法的修饰符为 public
    • 那么子类重写也只能用 public 修饰符

  • 异常可以缩小，但不能扩大

    • 比如父类方法抛出了一个范围比较小的异常，如 ClassNotFoundException
    • 子类重写就不能抛出比这个异常范围大的异常，如 Exception

泛型这个特性是从JDK 1.5才开始加入的，所以为了兼容之前的版本，泛型的实现采取了“伪泛型”的策略，即Java在语法上支持泛型，但是在编译阶段会进行所谓的“类型擦除”（Type Erasure），将所有的泛型表示（尖括号中的内容）都替换为具体的类型（其对应的原生态类型），就像完全没有泛型一样。

OOP，即Object-Oriented Programming

创建和初始化对象

通过 new 关键字创建对象，在此期间，除了分配内存空间之外，还会给创建好的对象进行默认的初始化以及调用类中的构造器

  • new 的本质就是调用构造器
  • 可以通过创建对象，调用这个类里的属性或者方法

什么是构造器？

构造器又称构造方法，是在进行对象的创建是必须调用的并且有两个特点

  • 必须和类名相同
  • 必须没有返回值类型，也不能写 void
  • 基于这两个特点可以得出为什么构造方法不能被继承

类默认会提供一个无参构造，但是一旦类中存在一个有参构造，那么无参构造必须 显示 定义

封装

隐藏对象属性的实现细节，控制对象属性的访问和修改权限，以一种特定的方式公开暴露给外部访问

  • 如使用 private 修饰实体类里的属性，利用 get()/set() 方法暴露给外部访问

继承

子类继承（通过 extends 关键词来实现继承）父类使其具有和父类相同的行为，本质就是都某一批类的抽象

java 中只能单继承，没有多继承，因为会产生二义性问题。但可以间接实现多继承

super 指代父类

  • 只能出现在子类的方法或者构造方法中
  • 只能写在代码第一行，且和 this 关键词互斥
  • 子类构造器会使用 super() 调用一次父类的构造方法且这行代码是 不可见 的

和 this 的区别

  • this 指代当前对象，super 代表父类的引用
  • this 不继承也可以使用，super 只能存在于有继承关系的类中
  • this 调用当前类的构造方法，super 调用的则是父类的构造方法

多态

关于java多态性之父类引用指向子类对象

一种行为具有多种形态和表现形式

  • 如重载，继承

多态是方法的多态，属性没有

  • 必须建立在继承关系或者其他关系上，否则会抛出异常 ClassCastException
  • 子类重写了父类的方法

  • 父类的引用指向子类的对象
    • 声明的是父类，实际指向的是子类的一个对象

  • 抽象类中可以没有抽象方法，但有抽象方法的类一定声明为抽象类
  • 抽象类，不能通过实例化（new）来创建对象，给子类继承
  • 抽象方法，只有声明，没有实现，由子类来实现
  • 子类继承抽象类，那么就必须实现抽象类中没有实现的抽象方法，否则该子类也必须声明为抽象类
  • 除了抽象方法，普通类有的抽象类都有

为什么要有抽象类？

抽象类是介于接口与普通类之间的一种结构，接口中定义的都是抽象方法，而类中都是普通方法。当我们存在一类事物，他们大多数结构和功能存在雷同，只有少部分存在不同，如果我们直接使用子类实现接口，那么结构和功能相同部分的代码将会造成大量冗余，因此我们需要一种结构，里面可以提供一部分已经实现了的共同的代码，剩下的一小部分由具体子类去实现即可，达到了代码的重用。

为什么抽象类不能实例化

  • 从Java规范来看：Java程序中规定了抽象类不能实例化。强行实例化就会报错。
  • 从功能角度来看：抽象类是一个结构和功能“不完整”的类，如果你让他强行“出生”的话，它可能会患有各种疾病，所以我们不能直接进行实例化，需要通过子类对结构和功能进行补全才能让它健康出生。
  • 从内存分配角度来看：抽象类只分配了在栈中的引用，没有分配堆中的内存。程序都有一个代码段，在内存中需要占据一定的内存，而抽象类没有具体的实现方法，无法具体的给它分配内存空间，所以为了安全，不JAVA不允许抽象类，接口直接实例化。

接口

普通类：只有具体实现

抽象类：具体实现和规范（抽象方法）都有

接口：只有规范，无法写实现方法（jdk8之后，可以定义默认方法），只做约束！

接口是一种规范，这里面定义了一系列规则，好比 法律 制定好之后都要去遵守。其本质就是契约

定义

  • 接口中定义接口其实都是抽象的 public abstract
  • 可以直接简写为 -> 返回值类型 接口方法名(形参)
  • 变量定义默认都是 public static final 修饰的常量

小结：

  • 接口是一种约束，定义了多个方法，让不同的实现类来实现
  • 接口中的方法默认都是 public abstract 修饰，属性都是 public static final 修饰的常量
  • 接口不能实例化，因为接口中没有构造方法
  • 接口通过 implements 可进行多实现，间接完成多继承
  • 实现类实现了接口，必须重写接口中的所有方法，除非是抽象类

对接口和抽象类的看法

抽象类和接口

接口更多的是赋予类某种能力（can-do）

抽象类则是将这一类事物的某个特征抽象出来，由具体的类来进行实现，或者将某一相同的行为抽象出来，减少代码重复（is-a）

Comparable 提供了一个比较的方法 compare ， User 类实现了这个方法，那么 User 这个类就具备比较功能，至于比较什么根据实际情况定，比如，User 里的年龄，身高等等

Phone 这个抽象类将手机的 尺寸和价格 这两个特征抽象出来，由具体的手机类，如 IPhone14PM、XIAOMI12SP 来描述其具体的尺寸是多大，价格是多少

手机都有一个共同行为就是充电，所有也可以将其抽象到 Phone 中以减少代码重复

内部类

成员内部类 && 静态内部类

  • 定义在普通类中
  • 可以访问类中的私有属性
  • 用 static 修饰的内部类就是静态内部类
    • 不可调用类中的私有属性及一些非静态的成员
    • 静态成员是属于类的，它加载的时候，非静态的成员还没有加载

局部内部类

  • 定义在方法中
  • 就类似 局部变量

匿名内部类

  • 没有名字去初始化类，不用将实例保存到变量中

异常

程序运行中出现的不期而至的各种状况

如：文件找不到，网络连接失败，非法参数…

Exception

  • 检查性异常

    • 编译期间就能确定的异常，不处理编译都不会通过，一般是别人导致的

    • 比如，这里会有一个 InterruptedException 如果不进行 try/catch 或者通过 throws 进行抛出，那么程序在编译时会报错

  • 运行时异常

    • 一般有程序逻辑错误引起的，应尽可能的避免
    • 这类异常不是检查异常，可在程序中选择性捕获，也可以不处理
    • 如：
      • ArrayIndexOutOfBoundsException -> 下标越界
      • NullPointerException -> 空指针
      • …

Error

  • Error 类对象由 Java 虚拟机生成并抛出，大多数错误与开发者所执行的操作无关，当这类错误发生时，JVM 一般会选择线程终止

  • 如：OutOfMemoryError，内存溢出

区别

  • Error 通常是致命性的错误，程序无法控制和处理，一般出现会导致程序崩溃
  • Exception 通常是可以被程序处理的，并且在程序中应该尽可能的去处理这些异常

异常捕获

通过 try/catch/finally 结构来捕获

也可以 使用 throw new 异常类型 手动抛出异常

或者将异常往外抛 throws 异常类型
