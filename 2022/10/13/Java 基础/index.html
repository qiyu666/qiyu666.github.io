java 简介

Java 三大版本

  • Java SE：标准版（桌面程序，控制台开发）
  • JavaME：嵌入式开发（手机，小家电）
  • JavaEE：E企业级开发（web端，服务器开发）

JDK、JRE、JVM

  • JDK（Java Development Kit）：Java核心。包含JRE和一些用于开发的工具包

  • JRE（Java Runtime Environment）：Java运行环境，包含JVM

  • JVM（Java Virtual Machine）：Java 虚拟机

编译型和解释型

  • 编译型，即先将整个源代码编译一遍，然后可以直接运行
  • 解释型，即将源代码翻译一行，执行一行，直至结束

Java 既是编译型又是解释型

因为所有的 Java 编写的代码在运行前都有一个预编译过程（从 .java 文件编译成 .class 文件），然后通过类加载器、字节码校验器、解释器等等运行在 JVM 上

方法

什么是方法

语句的集合，他们在一起执行一个功能

  • 方法是解决一类问题的步骤的有序组合
  • 方法包含于类或对象中
  • 方法在程序中被创建，在其他地方被引用

设计原则：其本意为功能块，用于实现某个功能的代码块的集合，设计时最好保持其原子性，即 一个方法只完成一个功能，利于扩展

方法定义

方法的语法结构

  • 修饰符，可选的，告诉编译器如何调用该方法，定义方法的访问类型
    • static：定义为静态方法，可直接通过类名进行调用
    • final：其修饰的方法，不可被重写
  • 返回值类型：方法可能会有返回值，void 表示没有返回值，无须使用 return 返回任意东西，此外，如 int String double... 都 必须 使用 return 返回对应类型的返回值
  • 方法名：方法实际名称，和参数列表共同构成方法签名
    • 参数列表：方法参数类型、顺序和参数个数
  • 参数类型：参数就好比一个占位符，当方法被调用时，必须传入对应的参数值类型的数据，当然这个参数是可选的，方法也可以没有参数
    • 形式参数：简称 形参 ，用于接收外界调用方法是传入的数据
    • 实际参数：简称 实参 ，调用方法时实际传入的参数
  • 方法体：定义方法具体功能

静态方法和非静态方法有什么区别

  • 静态方法不能重写，如果子类存在和父类相同的静态方法，那么父类这个方法是相对于子类所隐藏，非静态方法可以被重写

  • 静态方法调用只和左边定义的数据类型有关，而非静态方法不受影响

  • 静态方法属于类所有，类实例化前即可使用。非静态方法必须通过new出来的实例对象调用 🔴

  • 非静态方法可以访问类中的任何成员，静态方法只能访问类中的静态成员。且静态方法中不能调用非静态方法。

  • static内部只能出现static变量和其他static方法。而且static方法中还不能使用this、super关键字，因为它是属于整个类；

  • 静态方法效率上要比实例化高，但不能自动进行销毁，而实例化的则会在没有引用的情况下被GC回收；

  • 静态方法和静态变量创建后始终使用同一块内存，而使用实例的方式会创建多个内存，静态的东西都存在方法区中。

方法重载 && 方法重写

菜鸟教程 Java 重写(Override)与重载(Overload)

重载，即在一个类中，有相同名称的方法，但形参不同

重载规则

  • 方法名 必须相同
  • 参数列表 必须不同
  • 方法返回值类型可相同也可不同
  • 仅仅是返回值类型不同不足以成为方法的重载

重写，即子类对父类的允许访问的方法的实现过程进行重新编写

重写规则

外壳不变，核心重写！

  • 需要有继承关系

  • 只能重写 public/default 修饰的方法

  • 返回值、形参和方法名都 一定不能改变

  • 方法体重写

  • 修饰符可以扩大，但不能缩小

    • public > protected > default > private
    • 比如父类方法的修饰符为 public
    • 那么子类重写也只能用 public 修饰符

  • 异常可以缩小，但不能扩大

    • 比如父类方法抛出了一个范围比较小的异常，如 ClassNotFoundException
    • 子类重写就不能抛出比这个异常范围大的异常，如 Exception

通过 new 关键字创建对象，在此期间，除了分配内存空间之外，还会给创建好的对象进行默认的初始化以及调用类中的构造器

什么是构造器？

构造器又称构造方法，是在进行对象的创建时必须调用的，并且有两个特点

  • 必须和类名相同
  • 必须没有返回值类型，也不能写 void

基于这两个特点可以得出为什么构造方法不能被继承

类默认会提供一个无参构造，但是一旦类中存在一个有参构造，那么无参构造必须 显示 定义

抽象类

abstract 修饰的类是抽象类，修饰的方法是抽象方法

  • 抽象类中可以没有抽象方法，但有抽象方法的类一定声明为抽象类
  • 抽象类，不能通过实例化（new）来创建对象，给子类继承
  • 抽象方法，只有声明，没有实现，由子类来实现
  • 子类继承抽象类，那么就必须实现抽象类中没有实现的抽象方法，否则该子类也必须声明为抽象类
  • 除了抽象方法，普通类有的抽象类都有

为什么要有抽象类？

抽象类是介于接口与普通类之间的一种结构，接口中定义的都是抽象方法，而类中都是普通方法。当我们存在一类事物，他们大多数结构和功能存在雷同，只有少部分存在不同，如果我们直接使用子类实现接口，那么结构和功能相同部分的代码将会造成大量冗余，因此我们需要一种结构，里面可以提供一部分已经实现了的共同的代码，剩下的一小部分由具体子类去实现即可，达到了代码的重用。

为什么抽象类不能实例化

  • 从Java规范来看：Java程序中规定了抽象类不能实例化。强行实例化就会报错。
  • 从功能角度来看：抽象类是一个结构和功能“不完整”的类，如果你让他强行“出生”的话，它可能会患有各种疾病，所以我们不能直接进行实例化，需要通过子类对结构和功能进行补全才能让它健康出生。
  • 从内存分配角度来看：抽象类只分配了在栈中的引用，没有分配堆中的内存。程序都有一个代码段，在内存中需要占据一定的内存，而抽象类没有具体的实现方法，无法具体的给它分配内存空间，所以为了安全，不JAVA不允许抽象类，接口直接实例化。

接口

接口是一种规范，这里面定义了一系列规则，好比 法律 制定好之后都要去遵守。其本质就是契约

  • 接口中定义接口其实都是抽象的 public abstract
  • 可以直接简写为 -> 返回值类型 接口方法名(形参)
  • 变量定义默认都是 public static final 修饰的常量

小结：

  • 接口是一种约束，定义了多个方法，让不同的实现类来实现
  • 接口中的方法默认都是 public abstract 修饰，属性都是 public static final 修饰的常量
  • 接口不能实例化，因为接口中没有构造方法
  • 接口通过 implements 可进行多实现，间接完成多继承
  • 实现类实现了接口，必须重写接口中的所有方法，除非是抽象类
  • 可以定义默认方法和静态方法

类型擦除

Effects of Type Erasure and Bridge Methods

泛型不能是基本数据类型，因为 int，double 等等都不属于 Object 的子类

重载中如果出现泛型不一样的，不算重载

泛型这个特性是从JDK 1.5才开始加入的，所以为了兼容之前的版本，泛型的实现采取了“伪泛型”的策略，即Java在语法上支持泛型，但是在编译阶段会进行所谓的“类型擦除”（Type Erasure），将所有的泛型表示（尖括号中的内容）都替换为具体的类型（其对应的原生态类型），就像完全没有泛型一样。

Class 是来自 java.lang 包下的一个普通类，它保存了某个类或接口在运行时的所有信息(方法，注解，属性…)，将这些信息又抽象成一个个类，比如类的属性对应 Field 类，构造方法对应 Constructor 类…，可以在运行时，通过反射动态的获取类的属性或者执行其内部定义的方法

序列化是指将对象转换成字节数据的过程，然后可以将其保存到本地磁盘上永久保存，相反，反序列化就是将这些保存到本地磁盘的数据转回java对象的过程

在序列化时，如果被序列化的对象没有实现 Serializable 接口，会抛出 NotSerializableException 异常。

transient

被 transient 修饰的属性不参与序列化，在反序列化的时候这些没参与序列化的属性会根据其自身类型赋初始值，只能用于修饰属性

静态变量也不参与序列化，因为进行序列化的是实例对象，静态变量是属于类的，伴随着类的加载而加载

serialVersionUID

进行对象的序列化时会关联一个 serialVersionUID 版本号，然后在反序列化的时候通过这个版本号，来校验这个对象的输入输出方是否兼容。这个版本号如果在类中显示的定义了，那就以定义的为准，否则会根据类名、接口名、方法和字段生成一个序列号，如果这几个任意一个发生改变都会导致其 SUID 不一致，从而校验失败抛出 InvalidClassException 异常，所有强烈要求显示定义 SUID

IO流

文件操作

在 windows 操作系统中，文件路径使用的反斜杠来分隔，而反斜杠在 java 中表示转义符，所有在 java 中写文件路径，需要用两个反斜杠

File 类

来自 java.io 包中，可以用来，操作一个文件或者文件夹，比如获取文件名，删除文件，创建文件夹，创建文件等等

输入流，输出流

输入流，只能读取

输出流，只能写入

字节流，字符流

字节流，一个字节一个字节的读或写

字符流，一个字符一个字符的读或写，一个字符占两个字节

节点流，处理流

节点流，直接向IO设备读或写的流，称为节点流，好比连着花洒的管子

处理流，连接已经存在的流，对其进行处理的流，称为处理流，好比花洒
