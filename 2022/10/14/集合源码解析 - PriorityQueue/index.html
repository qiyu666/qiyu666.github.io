PriorityQueue 部分源码剖析

PriorityQueue 也是 Queue 的一种实现，即 优先队列，它能保证每次取出的元素都是队列中权值最小的。

PriorityQueue 是通过对 完全二叉树 排序实现的一个 小顶堆(任意一个非叶子节点的权值，都不大于其左右子节点的权值)，这也就意味着可以使用 数组 来存放元素。

扩容

PriorityQueue 扩容有几种情况

  • 如果原数组容量小于 64，那么将增加原来的 一倍+2
  • 如果大于 64，将增加原来的 1.5倍
  • 如果新容量超过了 MAX_ARRAY_SIZE，会调用 hugeCapacity() 来判断旧容量是不是也超过了 MAX_ARRAY_SIZE ，超过了那就直接取 Integer 的最大值，否则取 MAX_ARRAY_SIZE

添加元素

  • add(E e)：直接调用了 offer(E e) 进行添加

  • offer(E e)：往当前队列中添加元素。那不可避免会存在破坏 小顶堆 结构的可能，所以要进行调整。

    从上图可以很清楚的看到排序过程。先将目标元素添加到队列最后面，然后再调用 siftUp() 方法来维持小顶堆的特性。

    siftUp() 方法的原理就是，从 k 指定的位置开始，将 x 逐层与当前节点的 parent 比较。直到 compareTo() >= 0 为止，否则会交换 x 和当前节点的 parent 的位置

删除元素

  • remove()：这是父类 AbstractQueue 的方法，直接调用了 poll() 方法

  • remove(Object o)：这是来自 Collection 接口的方法

  • poll()：获取并删除第一个元素

    上述方法的区别在于，操作失败时，抛出异常 或者 返回 null

获取元素

  • element()：这是来自父类 AbstractQueue 的方法，直接调用了 peek() 方法

  • peek()：获取但不删除第一个元素

    上述两个方法的区别也是一样，在方法执行失败时，前者会 抛出异常，后者会 返回 null
